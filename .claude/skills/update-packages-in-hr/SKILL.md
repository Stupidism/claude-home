# Update @helloworld1812 Packages in workstream-hr

Run this skill from a workstream-hr worktree (the ticket's primaryRepo is `workstream-hr`).

## Step 1 — Read the ticket

Extract from the ticket body:

- The package name(s) to bump (e.g. `@helloworld1812/ws-components`). Default to "all" when the ticket says "bump all" or lists no specific package.
- The target version. If the ticket does not specify, use the latest version published to npm:
  ```bash
  npm view @helloworld1812/ws-components version
  ```
  All `@helloworld1812/*` packages are published together and share the same version.

## Step 2 — Update `package.json`

Edit `package.json` in the workstream-hr worktree. For each target package, replace the version in `dependencies` (and `devDependencies` if present). Preserve any existing caret/tilde prefix — only swap the numeric part.

When bumping every `@helloworld1812/*` package to the same version, prefer a single pass with `sed`:

```bash
NEW_VERSION="1.1800.0"   # replace with the actual version
sed -i '' -E "s|(\"@helloworld1812/[^\"]+\": \"\\^)[0-9]+\\.[0-9]+\\.[0-9]+\"|\\1${NEW_VERSION}\"|" package.json
```

Verify the diff lists only `@helloworld1812/*` lines with the new version:

```bash
git diff package.json
```

## Step 3 — Regenerate the lockfile

workstream-hr uses npm, not pnpm. Run:

```bash
nvm use && npm install
```

Confirm `package-lock.json` changed and no unrelated files were touched.

## Step 4 — Sanity checks

Skip a full build — it is slow. Instead verify the install resolved cleanly:

```bash
node -e "const v=require('@helloworld1812/ws-components/package.json').version; console.log(v)"
```

The printed version must match the target. Run lint and unit tests only for files that import the bumped package(s); do not run the full test suite for a version bump.

## Step 5 — Commit and PR

Use the standard commit + create-pr skills. Commit message should follow the conventional format:

```
chore(deps): bump @helloworld1812/ws-components to 1.1800.0 [WOR-XX]
```

PR title mirrors the commit. Body lists:

- The source PR in workstream-mono that released the new version.
- The packages bumped and their new version.
- A note that the bump was generated by the `update-packages-in-hr` skill.

## Scope guardrails

- Never bump unrelated dependencies in the same PR.
- Never edit source code — this skill only touches `package.json` and `package-lock.json`.
- If `npm install` fails with a peer-dependency conflict, stop and report in the workpad; do not add `--legacy-peer-deps` automatically.
